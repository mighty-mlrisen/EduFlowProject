# EduFlow

## О проекте

EduFlow — это полнофункциональная веб-платформа для публикации и чтения обучающих статей. Пользователи могут создавать авторские материалы с помощью встроенного редактора, взаимодействовать с контентом через лайки и комментарии, подписываться на авторов, сохранять понравившиеся статьи в избранное и отслеживать статистику своей активности. Платформа поддерживает разбивку статей по категориям, поиск по контенту и авторам, а также автоматическую суммаризацию статей с помощью дообученной NLP-модели.

---

## Основной функционал

- **Аутентификация** — регистрация, вход, выход, обновление JWT-токена
- **Статьи** — создание, редактирование и удаление статей; поддержка черновиков
- **Редактор** — WYSIWYG-редактор на базе TipTap: заголовки, списки, таблицы, блоки кода, изображения, ссылки, выравнивание текста
- **Суммаризация** — автоматическая генерация краткого содержания статьи с помощью дообученной русскоязычной T5-модели; результат кэшируется в Redis
- **Категории** — навигация и фильтрация статей по категориям
- **Лента** — главная лента всех статей и персональная лента из подписок
- **Поиск** — глобальный поиск по статьям и авторам
- **Лайки** — оценка статей
- **Комментарии** — комментарии к статьям с пагинацией
- **Избранное** — сохранение статей в персональный список
- **Подписки** — подписка на авторов и просмотр подписчиков
- **Профиль** — просмотр и редактирование данных профиля (имя, аватар, биография)
- **Статистика** — общая статистика платформы и персональная статистика автора
- **Тема оформления** — переключение между светлой и тёмной темой (сохраняется в localStorage)

## Инструкция по запуску

### Предварительные требования

- [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/) установлены и запущены

### Шаг 1 — Клонировать репозиторий

```bash
git clone <url-репозитория>
cd EduFlow
```

### Шаг 2 — Создать файл переменных окружения

Создайте файл `.env` в корне проекта (рядом с `docker-compose.yml`):

```env
# PostgreSQL
POSTGRES_DB=guidepedia_db
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_password

# Spring Boot datasource
DB_URL=jdbc:postgresql://postgres:5432/guidepedia_db

# JWT
JWT_SECRET=your_jwt_secret_key
JWT_SESSION_TIME=36000000
JWT_REFRESH_SESSION_TIME=86400000
```

> **Важно:** значения `POSTGRES_USER`, `POSTGRES_PASSWORD` должны совпадать в обеих секциях.

### Шаг 3 — Собрать и запустить контейнеры

```bash
docker compose up --build
```

При первом запуске Docker скачает базовые образы, соберёт JAR-файл бэкенда и собственный образ фронтенда. Это займёт несколько минут.

### Шаг 4 — Открыть приложение

| Сервис | Адрес |
|---|---|
| Приложение (фронтенд) | http://localhost |
| API (бэкенд) | http://localhost:8080 |
| Swagger UI | http://localhost:8080/swagger-ui.html |

### Остановка

```bash
docker compose down
```

Чтобы остановить и удалить тома с данными БД:

```bash
docker compose down -v
```

## Запуск в режиме разработки (без Docker)

### Backend

```bash
cd EduFlowAPI-service

# Убедитесь, что PostgreSQL запущен и доступен
# Задайте переменные окружения или отредактируйте application.properties

./mvnw spring-boot:run
# API будет доступен на http://localhost:8080
```

### Frontend

```bash
cd frontend

npm install

# Создайте файл .env.local
echo "VITE_API_URL=http://localhost:8080" > .env.local

npm run dev
# Приложение будет доступно на http://localhost:5173
```
