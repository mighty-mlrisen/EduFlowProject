# Тестирование: Суммаризация статей

Модуль проверяет генерацию краткого содержания статей с помощью дообученной T5-модели, работу кэша Redis и граничные случаи.

---

## TC-SUM-01 — Генерация саммари для IT-статьи

| Поле | Описание |
|---|---|
| **ID** | TC-SUM-01 |
| **Название** | Успешная генерация краткого содержания для полноценной IT-статьи |
| **Предусловие** | Опубликована статья длиной ≥ 300 слов по IT-тематике; кэш Redis для данной статьи пуст |
| **Шаги** | 1. Открыть страницу статьи <br> 2. Нажать кнопку «Получить краткое содержание» (или дождаться автоматической генерации) |
| **Ожидаемый результат** | ML-сервис возвращает саммари из 2–4 предложений на русском языке; текст саммари отражает основную суть статьи; отображается в отведённом блоке на странице статьи; в Redis создаётся запись `summary:article:{id}` с TTL 30 дней |
| **Статус** | Пройден |

## TC-SUM-02 — Получение саммари из кэша Redis

| Поле | Описание |
|---|---|
| **ID** | TC-SUM-02 |
| **Название** | Возврат ранее сгенерированного саммари из кэша без повторного инференса |
| **Предусловие** | Для статьи уже было сгенерировано саммари; запись `summary:article:{id}` присутствует в Redis |
| **Шаги** | 1. Открыть страницу той же статьи повторно <br> 2. Запросить краткое содержание |
| **Ожидаемый результат** | Саммари возвращается мгновенно (значительно быстрее, чем при первом запросе); ML-модель не вызывается повторно; содержание саммари идентично первому запросу |
| **Статус** | Пройден |

## TC-SUM-03 — Инвалидация кэша при обновлении статьи

| Поле | Описание |
|---|---|
| **ID** | TC-SUM-03 |
| **Название** | Сброс кэшированного саммари после редактирования статьи |
| **Предусловие** | Для статьи есть кэшированное саммари в Redis; пользователь является автором статьи |
| **Шаги** | 1. Отредактировать текст статьи и сохранить изменения <br> 2. Запросить краткое содержание статьи |
| **Ожидаемый результат** | Старый кэш удалён из Redis при сохранении статьи; при повторном запросе саммари генерируется заново на основе обновлённого текста; новое саммари отражает внесённые изменения |
| **Статус** | Пройден |

## TC-SUM-04 — Запрос саммари для слишком короткой статьи

| Поле | Описание |
|---|---|
| **ID** | TC-SUM-04 |
| **Название** | Обработка запроса саммари для статьи с недостаточным объёмом текста |
| **Предусловие** | Опубликована статья длиной менее 50 слов |
| **Шаги** | 1. Открыть страницу короткой статьи <br> 2. Запросить краткое содержание |
| **Ожидаемый результат** | Сервис возвращает информативное сообщение о том, что текст статьи слишком короткий для суммаризации; кнопка генерации неактивна или скрыта для коротких статей; ошибка обработана gracefully — приложение не падает |
| **Статус** | Пройден |

---

## TC-SUM-05 — Инвалидация кэша при удалении статьи

| Поле | Описание |
|---|---|
| **ID** | TC-SUM-05 |
| **Название** | Удаление записи саммари из Redis при удалении статьи |
| **Предусловие** | Статья с `id=42` существует; в Redis есть запись `summary:article:42`; пользователь является автором |
| **Шаги** | 1. Удалить статью через интерфейс <br> 2. Напрямую проверить Redis на наличие ключа `summary:article:42` (или убедиться, что повторный GET-запрос к несуществующей статье возвращает 404) |
| **Ожидаемый результат** | При удалении статьи бэкенд отправляет DELETE-запрос к Redis; ключ `summary:article:42` удалён; запрос саммари для несуществующей статьи возвращает корректный HTTP 404, а не устаревшие данные из кэша |
| **Статус** | Пройден |
