# Тестирование: Аутентификация

Модуль проверяет сценарии регистрации, входа, выхода и обновления JWT-токена.

---

## TC-AUTH-01 — Успешная регистрация нового пользователя

| Поле | Описание |
|---|---|
| **ID** | TC-AUTH-01 |
| **Название** | Успешная регистрация нового пользователя |
| **Предусловие** | Пользователь не зарегистрирован в системе; открыта страница регистрации |
| **Шаги** | 1. Ввести уникальный email (например, `test@example.com`) <br> 2. Ввести имя пользователя <br> 3. Ввести пароль длиной ≥ 6 символов <br> 4. Нажать кнопку «Зарегистрироваться» |
| **Ожидаемый результат** | Аккаунт создан; пользователь автоматически авторизован и перенаправлен на главную ленту; в localStorage сохранены access- и refresh-токены |
| **Статус** | Пройден |

---

## TC-AUTH-02 — Регистрация с уже занятым email

| Поле | Описание |
|---|---|
| **ID** | TC-AUTH-02 |
| **Название** | Регистрация с уже занятым email |
| **Предусловие** | В системе существует аккаунт с email `existing@example.com` |
| **Шаги** | 1. Открыть страницу регистрации <br> 2. Ввести email `existing@example.com` <br> 3. Заполнить остальные поля валидными данными <br> 4. Нажать «Зарегистрироваться» |
| **Ожидаемый результат** | Регистрация отклонена; отображается сообщение об ошибке «Пользователь с таким email уже существует»; перенаправления не происходит |
| **Статус** | Пройден |

---

## TC-AUTH-03 — Успешный вход в систему

| Поле | Описание |
|---|---|
| **ID** | TC-AUTH-03 |
| **Название** | Успешный вход зарегистрированного пользователя |
| **Предусловие** | Аккаунт с email `user@example.com` существует; пользователь не авторизован |
| **Шаги** | 1. Открыть страницу входа <br> 2. Ввести корректный email и пароль <br> 3. Нажать «Войти» |
| **Ожидаемый результат** | Пользователь авторизован; в localStorage записаны access-токен (TTL 10 ч) и refresh-токен (TTL 24 ч); выполнено перенаправление на главную страницу |
| **Статус** | Пройден |

## TC-AUTH-04 — Вход с неверным паролем

| Поле | Описание |
|---|---|
| **ID** | TC-AUTH-04 |
| **Название** | Вход с неверным паролем |
| **Предусловие** | Аккаунт с email `user@example.com` существует |
| **Шаги** | 1. Открыть страницу входа <br> 2. Ввести корректный email <br> 3. Ввести неверный пароль <br> 4. Нажать «Войти» |
| **Ожидаемый результат** | Авторизация отклонена; отображается сообщение об ошибке; токены не записываются; пользователь остаётся на странице входа |
| **Статус** | Пройден |

---

## TC-AUTH-05 — Автоматическое обновление access-токена через refresh

| Поле | Описание |
|---|---|
| **ID** | TC-AUTH-05 |
| **Название** | Автоматическое обновление истёкшего access-токена |
| **Предусловие** | Пользователь авторизован; access-токен истёк, refresh-токен действителен |
| **Шаги** | 1. Выполнить любой защищённый запрос к API (например, открыть страницу профиля) <br> 2. Axios-interceptor перехватывает ответ `401 Unauthorized` <br> 3. Автоматически отправляется запрос `POST /auth/refresh` с refresh-токеном |
| **Ожидаемый результат** | API возвращает новый access-токен; оригинальный запрос повторяется автоматически; пользователь не замечает прерывания сессии; UI обновляется без перезагрузки |
| **Статус** | Пройден |
